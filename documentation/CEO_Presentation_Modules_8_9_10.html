08 MCP Config Scanner
🔌

Model Context Protocol (MCP) Scanner

Discovers every AI tool your developers have connected to their IDEs and flags dangerous access rights before an incident occurs.

MODULE 08 · CONFIGURATION RISK

🎯 Business Problem

Developers connect AI assistants (Claude, Cursor, GitHub Copilot) to powerful tools — databases, filesystems, cloud APIs. Each connection is a potential security gap that no one is auditing.

✅ What This Module Does

Automatically reads every MCP configuration file on the system, lists all registered AI tool connections, detects embedded credentials, and scores each connection by risk level.

🔍 Where It Looks

Claude Desktop · Cursor IDE · VS Code / Insiders · Windsurf IDE · Kiro IDE · Any mcp.json found on disk (up to 4 levels deep)

🚨 What Gets Flagged

Unauthorized remote servers, embedded API keys/tokens, shell-level command execution, filesystem & database access, cloud API connections (AWS, Azure, GCP)

09 License Scanner & AST Parser
⚖️

License Scanner & AST Parser

Ensures every piece of code and every library your team uses is legally safe for enterprise deployment — protecting the company from copyleft and licensing lawsuits.

MODULE 09 · LICENSE GOVERNANCE · SEBI CSCRF ALIGNED

🎯 Business Problem

Developers often copy AI-generated code or import open-source libraries without checking licenses. A single AGPL or GPL dependency can force you to open-source your entire product.

✅ What This Module Does

Parses Python code at the AST (Abstract Syntax Tree) level — not just text search — to detect license headers, restricted library imports, and inline copyleft snippets across your entire codebase.

🔍 Files Scanned

Python (.py) — AST deep parse
JavaScript, TypeScript, Java, Go, Rust, C/C++ — regex inline scan
LICENSE, COPYING, LICENSE.md — workspace root files

🚨 What Gets Flagged

AGPL/GPL library imports, copyleft license headers in docstrings, Polyform Shield restrictions, Proprietary license mentions requiring legal review

10 Regulatory Compliance Scanner
🏛️

Regulatory Compliance Scanner

Automatically verifies that your AI infrastructure meets India's three key regulatory frameworks — DPDP Act, SEBI CSCRF, and CERT-In — without manual audit effort.

MODULE 10 · DPDP ACT · SEBI CSCRF · CERT-In

🎯 Business Problem

Indian regulators (SEBI, CERT-In, MeitY) are actively enforcing AI governance rules. A missed audit or unlawful data transmission can result in penalties, license revocation, or reputational damage.

✅ What This Module Does

Runs 3 automated compliance checks every scan — checking live network connections, SBOM baselines, and audit log integrity — and reports any regulatory gap instantly.

Step-by-Step Walkthrough

  1. User enters GitHub URL in the CLI menu (Option 3 → Custom Scan → GitHub Repository) or the Web UI dashboard
  2. Scanner calls GitHub's API (/zipball/HEAD) to download the repository as a ZIP archive — no git clone, no history
  3. ZIP is extracted to a system temp folder. An atexit hook guarantees cleanup even if the scan crashes
  4. ScanController launches with repo_mode=True and max_depth=10 — all 10 modules fire against the extracted folder
  5. Module 08 (MCP Scanner) looks for .cursor/mcp.json, .kiro/settings/mcp.json, and any other MCP configs inside the repo
  6. Module 09 (License Scanner) AST-parses every Python file and regex-scans JS/TS/Java/Go for license violations
  7. Module 10 (Compliance Scanner) checks for DPDP network risks, SBOM baseline, and audit log presence within the repo context
  8. All findings are aggregated, risk-scored, and written to report.json
  9. HTML report is generated — a full dark-theme interactive dashboard with tabbed views (Overview / Findings / Compliance)
  10. Temp folder is deleted — nothing from the scanned repo remains on disk
